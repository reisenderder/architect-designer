# PRAVILA.md: Системные правила и Технические стандарты проекта

## 1. Общие принципы взаимодействия агентов
*   **Приоритет Манифеста:** Любое техническое или текстовое решение должно сверяться с `PROJECT_MANIFEST.md`. Излишества (исраф) в коде или дизайне запрещены.
*   **Изоляция контекста:** Каждый агент выполняет только свою роль (Копирайтер пишет текст, Разработчик пишет код). Запрещено смешивать задачи.
*   **Компонентный подход:** Разработчик обязан дробить интерфейс на мелкие, переиспользуемые React-компоненты в Next.js.

## 2. Дизайн-система Tailwind CSS (Максимальный Минимализм)
Чтобы передать три главных качества (спокойствие, уверенность, облегчение), ИИ-разработчик должен использовать строго ограниченную палитру и сетку:

*   **Цвета (Спокойствие и Чистота):**
    *   `bg-slate-50` / `bg-white` — доминирующий фон (много свободного пространства, «воздуха» — облегчение).
    *   `text-slate-900` / `text-zinc-800` — глубокий, уверенный цвет для контрастного, легко читаемого текста.
    *   `brand-accent` (например, благородный приглушенный матовый зеленый или глубокий земляной оттенок, ассоциирующийся с надежностью дома, например `emerald-800` или `stone-700`) — используется крайне экономно (не более 5% интерфейса: только для фокуса на важных кнопках действия).
*   **Типографика (Уверенность и Точность):**
    *   Шрифты: Геометрические, чистые гротески без засечек (Sans-serif), передающие инженерную точность.
    *   Строгий контраст размеров: Большие, чистые заголовки с достаточным межстрочным интервалом (`leading-relaxed`).
*   **Пространство и Сетка:**
    *   Большие внутренние отступы (`py-16 md:py-24`, `space-y-12`) — интерфейс не должен быть перегруженным. Свободное пространство отражает скромность и отсутствие излишеств.

## 3. Стандарты разработки (Next.js App Router)
*   **Архитектура директорий:** Строгое разделение на `app/` (страницы и роутинг), `components/` (UI элементы) и `lib/` (утилиты/конфиги).
*   **Производительность:** Использовать Server Components (RSC) по умолчанию для максимальной скорости загрузки. Client Components (`'use client'`) — только для интерактивных элементов (квиз-калькулятор, переключатель тарифов).
*   **Чистота кода:** Никакого мертвого кода, четкая типизация (если используется TypeScript) или чистый, задокументированный JavaScript.

## 4. Контентные правила (Голос бренда)
*   **Запрет на «Восточные штампы»:** Запрещено использовать слова «роскошный», «элитный», «восточная сказка», а также узоры, арабески и золото в дизайне.
*   **Инженерная этика:** Вместо размытых фраз — точные формулировки. Например: вместо *«Мы делаем лучшие дома»* писать *«Проектируем раздельные зоны приватности и рассчитываем нагрузку инженерных сетей с точностью до миллиметра»*.
